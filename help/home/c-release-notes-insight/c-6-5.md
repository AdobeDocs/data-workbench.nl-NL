## Upgradevereisten {#section-84f8f38e9c424d7788c79146fce5a6fe}

Volg deze eisen en aanbevelingen wanneer het bevorderen aan Data Workbench 6.5.

* Door wijzigingen in het **`Components for Processing Servers\Communications.cfg`**-bestand moet u dit bestand bijwerken voor de DWB 6.5-release. De *SourceListServer*, *SegmentExportServer*, en *NormalizeServer* ingangen werden verwijderd. (DPU&#39;s moeten geen *sourcelist*, *segment export*, of *normalize servers* in werking stellen. )

## Systeemupdates {#section-bcdd732af53249b893d30bb42eeb14fc}

Deze functies zijn hernoemd, verwijderd of de installatiebestanden of mappen zijn geherstructureerd in deze release:

* Het Koord van de correlatie, de Matrijs van de Correlatie, het Koord van de Vereniging, de Matrijs van de Vereniging, de Score van de Volheid, en de beste visualisaties van de Attributie van de Aanpassing zijn nu **multi-pass visualisaties**, die hen toestaan om in de Server van het Rapport te lopen.

   >[!IMPORTANT]
   >
   >U kunt slechts één meervoudige visualisatie in een werkruimte hebben.
   >
   >Wanneer er meer dan één multi-pass visualisaties in een werkruimte zijn, zal de Server van het Rapport er niet in slagen rapporten door gebrek te produceren en de volgende fout te werpen:
   >
   >
   ```
   >Too many Multipass visualizations in workspace ....... (has #, 1 allowed).
   >```
   >
   >Vermijd deze fout door uw [!DNL ReportServer.cfg] dossier bij te werken of deze lijn toe te voegen aan uw bestaand dossier in *Rapporterend* sectie.   >
   >
   >
   ```
   >Max Multipass Per Slice = int: 1
   >```
   >
   >De waarde *Max Multipass Per segment* moet op 1 blijven staan (zie *Bekende problemen*).

* Wijzigingen aangebracht in bestanden in de map [!DNL Server\Menu\Admin].

   * Opzoeken verwijderd.
   * Toegevoegde archiefbestanden en gebruikersbestanden.
   * Gewijzigde bestanden, exportbestanden, logbestanden.

   Bijgewerkte bestanden zijn:

   ```
   Base\Menu\Admin\Order.txt (changes to menu)
   ```

   ```
   Base\Menu\Admin\Files.vw (removed Lookups and Events string, added 
   Archive and Users strings)
   ```

   ```
   Base\Menu\Admin\Archive Files.vw
   ```

   ```
   Base\Menu\Admin\User Files.vw
   ```

* Wijzigingen aangebracht in bestanden in de map [!DNL Base\Context\ServerDetails]:

   ```
   Base\Context\ServerDetails\Archive Files.vw
   ```

   ```
   Base\Context\ServerDetails\Export Files.vw
   ```

   ```
   Base\Context\ServerDetails\Log Files.vw
   ```

   ```
   Base\Context\ServerDetails\User Files.vw
   ```

   ```
   Base\Context\serverdetails\files.vw
   ```

   ```
   Base\workspaces\admin\dataset and profile\server files.vw
   ```

* Wijzigingen aangebracht in de [zelfbeheer van lidtoegang](/help/home/c-release-notes-insight/c-6-5.md).

   Een nieuw bestand in de pakketten *Base* en *Server* bevat een [!DNL User List.cfg] voor Admins om bij te werken voor lidtoegang.

## Vaste bugs {#section-3c47f9177e9d40d8938a5710399dc88f}

De volgende correcties zijn opvallend in Data Workbench 6.5 (sinds de vrijgave van Data Workbench 6.4).

* Correctie van de blokdecoderingsfout die werd geïntroduceerd in versie 6.4. Het probleem is opgetreden tijdens de verwerking van een zeer grote bezoeker.
* Correctie van uitlijningsfout geïntroduceerd met compilerupgrade. Probleem met invloed op replicatie in release 6.4

## Bekende problemen {#section-350ae121ca3442babbef4d0a2b027df1}

* Het gebruik van *Windows-certificaatarchief* is beperkt tot alleen het Engels (geen ondersteuning voor de Chinese of Japanse taal).
* *In* regressieanalysegrafieken worden nieuwe werkruimteselecties niet herkend. Een mogelijke oplossing is om de regressie na het maken van een nieuwe selectie opnieuw uit te voeren.
* De waarde *Max Multipass Per segment* moet op 1 blijven staan. U kunt niet meer dan één van de volgende visualisaties in één enkele werkruimte hebben voorgenomen voor de Server van het Rapport: Volhedensescore, kenmerk Best Fit, Associatiematrix, Associatiekoord, Matrix Correlatie of Koord Correlatie.
