# Data Workbench 6.4 Opmerkingen bij de release{#data-workbench-release-notes}

Opmerkingen bij de release van Data Workbench 6.4 omvatten nieuwe functies, upgradevereisten, opgeloste problemen en bekende problemen.

Als u vorige functies en oplossingen voor eerdere versies wilt bekijken, raadpleegt u het archief met [opmerkingen bij de release](https://docs.adobe.com/content/help/en/data-workbench/using/release-notes/release-notes.html).

## Apparaataturen met cache in geheugen{#device-atlas-with-in-memory-cache}

Het bestand DeviceAtlas.bundle gebruikt nu een cache in het geheugen om de prestaties van zoekopdrachten aanzienlijk te verbeteren

Door gebrek, zal de Atlas van het Apparaat tot 100.000 gebruiker-agenten en hun eigenschappen in het voorgeheugen onderbrengen. De LRU-cache bevindt zich volledig op zichzelf in de DeviceAtlas.bundle. Elke versie van de server die een bundelbestand kan gebruiken, profiteert automatisch van de verbeterde prestaties zodra het nieuwe bestand is geladen.

**Het bestand DeviceAtlas.cfg wijzigen**

De maximumgrootte van de LRU-cache kan worden geconfigureerd door de parameter Limit voor in cache geplaatste elementen in het bestand DeviceAtlas.cfg te wijzigen.

Vanaf Data Workbench 6.4 kunt u de standaardlimiet voor in cache geplaatste elementen van &quot;100000&quot; overschrijven door de waarde ervan te wijzigen en het bestand op te slaan. De standaardwaarde is geselecteerd om voldoende te zijn voor de meeste behoeften.

```
component = DeviceAtlasComponent: 
  DeviceAtlas Bundle File = string: Lookups\\DeviceAtlas\\DeviceAtlas.bundle 
  Unsynchronized Bundle Extraction Path = string: Temp\\DeviceAtlas\\ 
  
<b> Cached Elements Limit = unsigned int: 100000</b> 
```

## Upgradevereisten {#section-84f8f38e9c424d7788c79146fce5a6fe}

Volg deze eisen en aanbevelingen bij de upgrade naar Data Workbench 6.4.

>[!IMPORTANT]
>
>U wordt aangeraden de nieuw geïnstalleerde standaardconfiguratiebestanden te gebruiken en deze aan te passen in plaats van bestanden van een vorige installatie te verplaatsen, met de volgende uitzonderingen:

* **Voeg** ***Uitgesloten Processen*** voor de Bescherming van het Eindpunt van het Centrum van het *Systeem van MS in Vensters 2012 Servers* voor de volgende uitvoerbare lijnen toe:

   * **`InsightServer64.exe`**
   * **`ReportServer.exe`**
   * **`ExportIntegration.exe`**

   Dit zal wit-lijst rechten voor deze interface uitvoerbare voorwerpen toestaan.

* **Werk het *`Trust_ca_cert.pem`*certificaat bij op de servers**.
* **Reorganisatie van kenmerkprofielen**.

   * De naam van de map *Attribution* is gewijzigd in ***Attribution - Premium*** (in de standaardinstallatie van *Profiles* \ *Attribution - Premium*).

   * Het *Premium* -profiel is verwijderd en de werkruimte is verplaatst naar de nieuwe map ***Attribution - Premium*** .

* **Werk *Attribution-Premium*-instellingen** bij. Als u aangepaste profielen hebt met parameterinstellingen die het standaardprofiel voor *Adobe SC* overschrijven, moet u de aangepaste velden in deze configuratiebestanden bijwerken:

   * **`Decoding Instructions.cfg`**
   * **`SC Fields.cfg`**

* Vanwege deze reorganisatie wilt u de oude mappen *Attribution* en *Premium* verwijderen uit de serverinstallatie.

   **Deze instellingen wijzigen**

   ```
   Profile = profileInfo:  
     Active = bool: true 
     Directories = vector: 6 items 
       0 = string: Base\\ 
       1 = string: Geography\\ 
       2 = string: Predictive Analytics\\ 
       3 = string: Adobe SC\\ 
       4 = string: Attribution\\ 
       5 = string: Premium\\
   ```

   op deze instellingen:

   ```
   Profile = profileInfo:  
     Active = bool: true 
     Directories = vector: 5 items 
       0 = string: Base\\ 
       1 = string: Geography\\ 
       2 = string: Predictive Analytics\\ 
       3 = string: Adobe SC\\ 
       4 = string: Attribution - Premium\\
   ```

* **Werk aangepaste bestanden** Meta.cfg bij (indien nodig).

   De **`Meta.cfg`** bestanden in **`Base\Context and AdobeSC\Context`** mappen zijn bijgewerkt in deze release.

   Als u het bestand **meta.cfg** tijdens de installatie overschrijft, moet uw profielkopie worden bijgewerkt met deze parameters en moet de juiste **metagegevensvector** worden ingevoerd:

   ```
   94 = meta: 
     path = string: SegmentExport:CRS Configuration/CRS Attributes 
     acceptable children = vector: 1 items 
       0 = Template: 
         name = string: CRS Attributes 
         value = CRSAttributeConfiguration: 
           Attribute Name = string: 
           Attribute Type(int,string) = string: 
           Field Name = string: 
   
   95 = meta: 
     path = string: SegmentExportQuery:CRS Configuration/Report Suite 
     acceptable children = vector: 1 items 
       0 = Template 
         name = string: Add Report Suite 
         value = string:
   ```

* **Stel Rapportservermachtigingen** in om Microsoft Excel-rapporten te genereren op Windows 2012-servers.

   1. Plaats toestemming van de wortelomslag (**`E:\ReportServer\`**) aan *iedereen = volledige controle*.
   1. Maak de volgende mappen met de juiste machtigingen:

      ```
      C:\Windows\SysWOW64\config\systemprofile\AppData\Local\Microsoft\Windows\INetCac‌he 
      C:\Windows\System32\config\systemprofile\AppData\Local\Microsoft\Windows\INetCac‌he 
      C:\Windows\System32\config\systemprofile\Desktop 
      C:\Windows\SysWOW64\config\systemprofile\Desktop
      ```

      >[!NOTE]
      >
      >Als u de Server van het Rapport op de Server 2012 van Vensters in werking stelt, moet u de Server 2012 R2 hebben van Vensters geïnstalleerd.

   1. Wijs &quot;SYSTEM&quot;als eigenaar voor deze omslagen toe.

* **Voeg doopvonten aan de Server van het Rapport toe.** Voeg in het bestand **`ReportServer.cfg`**deze lettertypen toe (voor alle talen):

   ```
   Fonts = vector: 3 items 
     0 = string: Arial 
     1 = string: SimSun 
     2 = string: MS Mincho
   ```

* **Werk uw versie van Microsoft Excel** bij (indien nodig).

   Met de versie van Data Workbench 6.4, is de steun voor Excel 2007 gestopt. Ook omdat de Data Workbench slechts op Microsoft Windows voor architectuur met 64 bits loopt, adviseert men dat u ook een versie met 64 bits van Microsoft Excel installeert.

* **64-bits architectuur** vereist voor Workstation (client)-installatie.
* **Voer de wizard** Workstation instellen uit.

   Installeer de nieuwe versie van het werkstation (cliënt) door ***InsightSetup.exe*** te downloaden en te lanceren en door de opstellingsinstructies te stappen. De installatiewizard installeert uw bestanden standaard op een nieuwe locatie:

   Programmabestanden worden nu standaard opgeslagen in:

   ```
   C:\Program Files\Adobe\Adobe Analytics\Data Workbench
   ```

   Gegevensbestanden (profielen, certificaten, traceringslogbestanden en gebruikersbestanden) worden nu standaard opgeslagen naar:

   ```
   C:\Users\<username>\AppData\Local\Adobe\Adobe Analytics\Data Workbench\
   ```

* **Voeg lettertypen toe aan het werkstation**.

   Voeg in het **`Insight.cfg`** bestand de volgende lettertypen toe (voor alle talen):

   ```
   Fonts = vector: 3 items 
     0 = string: Arial 
     1 = string: SimSun 
     2 = string: MS Mincho
   ```

## Systeemupdates {#section-8e6b9dcaf333453bb2079b151bbbce93}

Deze functies zijn hernoemd, verwijderd of de installatiebestanden of mappen zijn geherstructureerd in deze release:

* De map **Base.zip** is niet meer opgenomen in het updatepakket voor de versie.
* Het **bestand DeviceAtlas.bundle** gebruikt nu een cache in het geheugen om de prestaties van zoekopdrachten te verbeteren.
* In het bestand **Log Processing.cfg** is de parameter *Chunk Size* onder *Log Sources* verwijderd.
* In het bestand **Disk Files.cfg** is de parameter *Detect Disk Corrupt* verwijderd op de volgende locaties:

`\server\components\disk files.cfg`

`\server\components for processing servers\disk files.cfg`

* Nieuwe de dienstbeschrijvingen voor de Diensten *van* Adobe Analytics Premium en voor de Diensten *van het Rapport van* Adobe Analytics Premium in de uitvoerbare eigenschappen.
* De functie *Master marketingprofiel exporteren* in de tabel Details is hernoemd naar ***Profielen en soorten publiek exporteren***.
* De functie *Testen en exporteren* doel in de tabel Details is hernoemd naar ***Adobe Target Export***.

## Vaste bugs {#section-bcdd732af53249b893d30bb42eeb14fc}

De volgende correcties zijn aangebracht in Data Workbench 6.4 (sinds de release van Data Workbench 6.31).

* De volheidsscore werd niet opnieuw ingesteld wanneer verschillende invoer in dezelfde werkruimte werd uitgevoerd. Dit wordt nu op de juiste wijze opnieuw ingesteld.
* Er zijn geen telbare afmetingen beschikbaar wanneer de Correlatie-matrix voor het eerst wordt geopend.
* De uitvoer van de segmenten van het Doel mislukte omdat het mboxPC- gebied ontbrak. Dit is nu opgelost.
* ID-aanvraag is correct opgemaakt. Door het gebruik van de mbox3rdpartyId-identificatie in plaats van standaard-PCID&#39;s weigerde Adobe Target aanvragen die via de integratie Doel/Data Workbench (met de **`ExportIntegration.exe`**) werden gegenereerd. Deze id-aanvraag wordt nu correct opgemaakt en de doorvoer is voltooid.
* Het geheugenlek van de rapportserver tijdens het exporteren naar Excel is opgelost.

## Bekende problemen {#section-d0cd72e82d1444a2b069dfcda5d7692a}

De volgende beperkingen zijn bekend in Data Workbench 6.4.

* **`ExportIntegration.exe`** vereist opdrachtregelargumenten in het Engels. De naam van het uitvoerbestand moet in het Engels worden genoemd voor *Adobe Target Export*, *Profiles and Audiences Export* en *Customer Record Service Export*.
* In de *Profielen en de Uitvoer* van de Publiek, die onbevoegde karakters (`[CR]` of `[TAB]`) als kolomnamen ingaan produceert onjuiste logboeken resulterend in gegevens niet correct uitvoeren.
* In Chinese en Japanse versies kan het probleem van Unicode-tekencodering worden aangetroffen in de padbrowser.
